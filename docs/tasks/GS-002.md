## Overview
Authorization middleware to check logged in user role and permissions before accessing getList of classes

**Type**: Feature

## Context
- If logged in user is an admin, he/she can get list of all classes
- If logged in user is a teacher, he/she can get list of classes that he/she is teaching.
- If logged in user is a student, he/she can see only one class that he/she is in
- If logged in user is not one of the above roles, he/she cannot access API getList of classes

## Target | Goal
- Middleware service that get token from the RESTful API, sent by FE, return the role, scopes/permissions/policies of user
- Implement new domain `classes` handle the API `GET /classes` and `GET /classes/{classId}`
- Implement unit test, integration test
- Hanle errors: 401, 500 (if the classId does not exist in DB, still return 401 - Unauthorized)
