## Overview
Implement validators, custom validators for checking rules from yaml files of requestBody of creating classes.

**Type**: Feature

## Context
When developers prepare yaml files to define a REST API, they can use the x-oapi-codegen-extra-tags:validate for detail validation rules

```yaml
type: object
properties:
    name:
        type: string
        description: Name of the class
        x0oapi-codegen-extra-tags:
            validate: "required, unique_in:classes_table:name"
```

and the "unique_in" will be detected by the validator in Go, and then it will check the table name as "classes_table", field "name":

`validator.go`
```go
package rules

func (v *Validator) registerCustomValidations() {
    if err := v.validate.RegisterValidationCtx("unique_in", v.UniqueInValidator); err != nil {
        logger.Errorf("Failed to register unique_in validation: %v", err)
    }
}
```

`unique_in.go`
```go
package rules

func (v *Validator) UniqueInValidator(ctx context.Context, fl validator.FieldLevel) bool {
    // handle the validation logic here
}
```

## Target | Goal
- New directory ffor rules and validator
- validator will deffine, register all rules
- each rule will be a separated file `<rule_name>.go`
- the create new class with name and ID where ID, and name of class must be unique
- updating class name must check the new name does not existing in the classes table, except the current name that is going to be changed => check if the ID is not the same as updating class ID, and name is the same new name, then return false for the validator
